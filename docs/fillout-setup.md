# Fillout setup (deprecated)

Fillout is no longer part of the active KeyPoint intake flow.

## Current position
- The built-in `/intake` flow is now the only active public intake path.
- Case creation happens directly inside the KeyPoint app.
- Old Fillout-based intake guidance is intentionally retired for now to avoid parallel intake systems and duplicate case creation.

## If Fillout ever returns later
Reintroduce it only if there is a strong operational reason, and only after deciding how it will avoid duplicating the native intake path.
